### 自习模式 v1.5（新增第 10 学生页面）

**位置**: `frontend/src/app/zixi/page.tsx`

**核心特性**：
- **三状态机**：`idle`（配置页）→ `running`（横排运行）→ `report`（报告页）
- **摄像头本地姿态检测**：TF.js + MoveNet Lightning（Google 预训练，~3MB），CPU 推理 ~30ms/帧
- **零后端改动**：复用现有 `POST /api/v1/evaluation/record` 端点，写 3 种新 action 到 `learning_records` 表
  - `study_session_start` — 用户开始自习时
  - `study_patrol` — 每次姿态判定（30~120s 间隔，由 difficulty 控制）
  - `study_session_end` — 用户结束/到时间时（含汇总统计）
- **数据联动**：现有 `scheduled_analysis_service`（每 4h）自动检测新记录，调用 `behavior_analysis_agent` 更新 `student_profiles.dimensions.focus`（专注力维度）
- **物理摄像头智能过滤**：自动枚举并排除 Iriun / OBS Virtual / DroidCam 等虚拟摄像头
- **隐私优先**：摄像头画面完全在浏览器处理，不上传任何图像

**三档巡查频率**：
- 宽松（easy）：120s — 适合长自习
- 标准（normal）：60s — 默认
- 严格（strict）：30s — 高强度模式

**行为判定**（3 个关键点：鼻子 + 左肩 + 右肩）：
| 标签 | 触发条件 | 含义 |
|---|---|---|
| `focus` | 鼻尖高于肩中点 | 专注 |
| `distracted` | 鼻尖低于肩中点 > 0.5 倍肩宽 | 低头 / 看桌面 |
| `absent` | 关键点缺失 / 置信度 < 30% / 肩宽 < 10px | 离席 |

**离席防误报**：连续 3 次 absent 才记录一次（避免转头瞬间误报）

**新增文件**：
- `frontend/src/hooks/useCameraPatrol.ts` — TF.js + MoveNet + 行为判定核心 hook
- `frontend/src/components/CameraToggle.tsx` — 受控摄像头开关组件
- `frontend/src/app/zixi/page.tsx` — 3 状态机自习主页

**修改文件**：
- `frontend/package.json` — +`@tensorflow/tfjs@^4.20.0` +`@tensorflow-models/pose-detection@^2.1.3`
- `frontend/src/app/globals.css` — +`study-*` 系列样式
- `frontend/src/components/layout/Sidebar.tsx` — +Focus 组 /zixi 入口
- `backend/app/api/dashboard.py` — +8 行（study_sessions 本周计数）

**横排运行布局**（区别于其他页面）：
- 左侧 340px 摄像头视频（4:3，含 LIVE 角标 + 关闭按钮 + 状态浮层）
- 右侧倒计时（240px 圆环 + 元信息三联：巡查/专注/低头）

**idle 16:9 预览**：开启摄像头后立即显示实时预览 + "摄像头已就绪"角标 + "仅本地处理"脚标

**报告页**：3 张统计卡（时长/巡查次数/专注率）+ 5 行明细（完成状态/专注时长/低头次数/离席次数/最长连续专注）+ 巡查时间线（hover 显示 `#N 专注 X%` tooltip）

**兼容性**：现代浏览器（Chrome 88+ / Edge 88+ / Firefox 89+ / Safari 15+），getUserMedia 需要 HTTPS 或 localhost
