## 一、项目定位

**智枢 (SmartHub)** 是第十五届中国软件杯 A3 赛题作品，定位为**多智能体个性化学习资源生成系统**。

| 项 | 内容 |
|---|---|
| 赛题 | A3：基于大模型的个性化资源生成与学习多智能体系统 |
| 出题方 | 科大讯飞 |
| 课程切入点 | 人工智能导论 |
| 目标用户 | 高等教育学生（计算机 / AI 专业本科） |
| 核心命题 | 让每个学生拥有专属 AI 学习助手 |
| 评分项 | F1 对话式画像 35% + F2 多智能体资源生成 45% + F3 路径规划 + F4/F5 加分 |

## 二、业务闭环

### 2.1 核心闭环五步走

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ ① 对话  │ →  │ ② 画像  │ →  │ ③ 资源  │ →  │ ④ 路径  │ →  │ ⑤ 评估  │
│  输入   │    │  构建   │    │  生成   │    │  规划   │    │  反馈   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
     ↑                                                      │
     └──────────────── ⑥ 持续优化 ─────────────────────────┘
```

### 2.2 各步骤详解

| 步骤 | 输入 | 处理 | 输出 | 涉及 Agent |
|---|---|---|---|---|
| ① 对话输入 | 自然语言 | 路由分类 + RAG 检索 | 流式回答 | Tutor / Chat |
| ② 画像构建 | 对话 + 行为 | 7 维评估 + 规则更新 | JSONB 画像 | Initial Assessment + Behavior Analysis |
| ③ 资源生成 | 知识点 | 多 Agent 并行生成 | 知识/代码/导图/题 | Document / MindMap / Exercise |
| ④ 路径规划 | 知识点 + 画像 | DAG + 拓扑排序 | 学习路径 + 每日计划 | Learning Path / Path |
| ⑤ 评估反馈 | 学习行为 | LLM 报告 + 趋势 | 评估报告 | Evaluation + Scoring |

## 四、核心业务流程

### 4.1 主链路：学生首次使用到熟练使用

```
注册/登录 → 对话式画像构建 → 查看看像 → AI 生成资源 → 学习计划
    ↓                                                      ↓
答题(题库) ← 错题自动收录 ← 答错时                        节点完成
    ↓                                                      ↓
错题分析(AIBackend) → 同类题推荐 → 掌握度提升          综合测试
    ↓                                                      ↓
自习模式(摄像头) → 专注力数据 → 画像更新              评估报告
    ↓
持续优化画像 → 个性化推荐更精准
```

### 4.2 子链路：多智能体资源生成（F2 重点 45%）

```
用户输入"讲解 A* 算法"
   ↓
Master Agent (StateGraph 10 节点编排)
   ↓
intent_recognition (意图识别) → task_planning (任务规划)
   ↓
conditional_route (条件路由)
   ↓
6 个 Agent 并行/串行执行:
   ├─ DocumentAgent    → 知识讲解 Markdown
   ├─ MindMapAgent     → Mermaid 思维导图
   ├─ ExerciseAgent    → 练习题（dual-format 流式）
   ├─ AudioAgent       → 音频脚本（可选）
   ├─ LearningGuideAgent → 学习指引
   └─ ResourceCreatorAgent → 资源创建编排
   ↓
防幻觉三层验证 (PatternDetector + SourceValidator + LLMValidator)
   ↓
result_aggregation → response_generation
   ↓
SSE 流式返回给前端 → 实时渲染
```

### 4.3 画像联动双路径（创新点）

```
┌──────────────────────────────────────────────────────────────┐
│ 规则引擎（apply_rule_updates · 即时/高频/确定性）              │
│   ├─ tiku 每题  → application（正确率映射）                  │
│   ├─ zixi 学习  → memory + focus（时长映射）                  │
│   ├─ 节点完成   → learning_goal（+2/节点，全完成 +5）         │
│   ├─ 评估报告   → comprehension + knowledge_base             │
│   ├─ 错题回顾   → memory（答对+1/掌握+3/答错-1）            │
│   └─ 思维导图   → imagination（每次+1）                     │
├──────────────────────────────────────────────────────────────┤
│ AI 分析（analyzeBehavior · 批量/低频/LLM 深度）              │
│   ├─ tiku 每 5 题 → 全 7 维 AI 推断                          │
│   ├─ zixi 结束    → 全 7 维 AI 推断                          │
│   ├─ resources 学习 → analyzeBehavior('study')                │
│   ├─ duihua 对话   → analyzeBehavior('chat')                 │
│   └─ 定时分析      → 每 4 小时 scheduled_analysis_service     │
└──────────────────────────────────────────────────────────────┘
                            ↓
              profile_service.apply_llm_updates
                            ↓
        flag_modified + pg_try_advisory_lock 持久化
                            ↓
                  画像缓存 5min（行为后自动清）
```

## 六、多智能体编排（StateGraph 10 节点）

### 6.1 编排流程

```
intent_recognition        # 1. 意图识别（关键词快路由 + LLM）
       ↓
task_planning            # 2. 任务规划（分解为子任务）
       ↓
conditional_route        # 3. 条件路由（分发到对应 Agent）
       ↓
┌─────────────────────────────────────────────┐
│ 6 个 Agent 并行/串行执行                       │
│  ├─ document_agent       知识讲解              │
│  ├─ mindmap_agent        思维导图              │
│  ├─ exercise_agent       练习题（dual-format） │
│  ├─ path_agent           路径规划              │
│  ├─ tutor_agent          智能问答              │
│  └─ audio_agent          音频脚本              │
└─────────────────────────────────────────────┘
       ↓
result_aggregation        # 4. 结果聚合
       ↓
response_generation       # 5. 响应生成
```

### 6.2 15 个 Agent 模块清单

| Agent | 功能 | 评分项 |
|---|---|---|
| Master Agent | StateGraph 编排器 | F2 核心 |
| Initial Assessment Agent | 7 维画像评估 | F1 |
| Document Agent | 知识讲解 | F2 |
| Exercise Agent | 自适应出题 | F2 + 错题本 |
| Path Agent | DAG 学习路径 | F3 |
| Learning Path Agent | AI 自主拆分路径 | 学习计划 |
| Learning Guide Agent | 结构化学习指引 | 学习计划 |
| Tutor Agent | RAG 智能问答 | F4 |
| MindMap Agent | Mermaid 思维导图 | F2 |
| Audio Agent | 音频脚本生成 | F2 |
| Behavior Analysis Agent | 行为分析 + 画像更新 | F1 |
| Coordinator Agent | 任务协调 | 内部 |
| Review Agent | 质量审核 | 防幻觉 |
| Resource Creator Agent | 资源创建编排 | F2 |
| Wrong Question Agent | 错因分析(4 步思考链) | 错题本 |
| Scoring Agent | AI 智能评分 | 综合测试 |

## 七、技术栈与协议

### 7.1 前端

| 技术 | 版本 | 用途 |
|---|---|---|
| Next.js (App Router) | 14.2.5 | 框架 |
| TailwindCSS | 3.4 | 样式（混合自定义 CSS 变量） |
| TypeScript | 5.x | 类型 |
| 本地字体 woff | - | Inter / Newsreader / JetBrains Mono |
| **@tensorflow/tfjs** | 4.22 | 自习模式姿态检测 |
| **@tensorflow-models/pose-detection** | 2.1 | MoveNet Lightning |

### 7.2 后端

| 技术 | 版本 | 用途 |
|---|---|---|
| FastAPI | 0.136 | Web 框架 |
| SQLAlchemy | 2.0 async | ORM |
| asyncpg | - | PG 异步驱动 |
| LangGraph | - | StateGraph 编排 |
| Celery | - | 定时任务（每日 4 点预生成评估） |

### 7.3 LLM 三客户端

| 客户端 | 鉴权 | 用途 | 启用环境 |
|---|---|---|---|
| **MiniMax-M3** | `Authorization: Bearer` | **当前默认** | 开发测试 |
| MiMo v2.5 | `api-key` 头 | 备选 | 切换 `LLM_PROVIDER=mimo` |
| **讯飞星火 V4** | `Authorization: Bearer` | **比赛必用** | 切换 `LLM_PROVIDER=spark` |

### 7.4 协议亮点

| 协议 | 说明 | 作用 |
|---|---|---|
| **dual-format 流式** | LLM 一份输出 = markdown + `---JSON_DATA---` + JSON | 题目逐行实时渲染（不是伪流式） |
| **SSE 真流式** | 9 个真流式端点 | 实时返回生成结果 |
| **三层防幻觉** | PatternDetector + SourceValidator + LLMValidator | 确保生成内容有据可查 |
