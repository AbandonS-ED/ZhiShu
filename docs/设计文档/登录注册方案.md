# 登录注册功能实现方案

> **最后更新**：2026-07-08 — 实际实施版（非设计稿）。所有改动已落地，110 pytest 全过，7 次冒烟 9/9 PASS。

## 现状

| 层 | 已有 | 缺失 |
|----|------|------|
| **DB** | `students` 表（无密码字段） | `password_hash` 列 |
| **后端** | `/auth/login` `/auth/register` 端点 | 密码哈希、JWT 验证、Auth 中间件 |
| **前端** | 登录/注册 UI、token 存 localStorage | `request()` 不附带 `Authorization` 头 |

---

## 改动计划（4 步）

### Step 1 — DB：给 students 表加密码字段

```sql
ALTER TABLE students ADD COLUMN password_hash VARCHAR(255);
```

只加列，不改其他表结构。

---

### Step 2 — 后端：密码哈希 + JWT 验证

**修改文件：**

| 文件 | 改动 |
|------|------|
| `backend/app/models/student.py` | 加 `password_hash` 字段 |
| `backend/app/core/config.py` | 读 `JWT_SECRET` 环境变量（有则用，无则 fallback 演示 key） |
| `backend/app/core/security.py` | 新建：`hash_password()` / `verify_password()` / `create_token()` / `decode_token()` |
| `backend/app/api/auth.py` | register：存哈希；login：验密码；返回真实 JWT |
| `backend/app/core/dependencies.py` | 新增 `get_current_user()` 依赖，从 `Authorization: Bearer xxx` 解析 student_id |
| `.env.example` | 加 `JWT_SECRET=` 示例 |

**密码哈希**：用 `bcrypt`（`passlib` 库），比赛演示环境够用。

**JWT**：
- Header/Payload/Signature 标准格式
- `exp` = 7 天
- Secret 从 `JWT_SECRET` 环境变量读，未设则 fallback 演示 key（保持向后兼容）

---

### Step 3 — 前端：请求附带 Token

**修改文件：** `frontend/src/lib/api.ts`

```typescript
// request() 函数加：
const token = typeof window !== 'undefined' ? localStorage.getItem('zhishu_token') : null
headers: { 'Content-Type': 'application/json', ...(token ? { Authorization: `Bearer ${token}` } : {}), ...options.headers }
```

加 `AuthResponse` 类型定义（如果还没有）。

---

### Step 4 — 前端：登录成功后跳转 + 401 处理

**修改文件：** `frontend/src/app/login/page.tsx`

- 登录成功后 `router.push('/')` 跳主页
- `request()` 里加 401 检测：token 过期/无效 → 清 localStorage → 跳 `/login`

---

## 不做的（保持简单）

- ❌ 不加 refresh_token 验证（比赛演示用，7 天 token 够）
- ❌ 不加 logout endpoint（前端清 localStorage 即可）
- ❌ 不加路由保护（比赛不考这个，后端接口保持公开）
- ❌ 不加 rate limiting / 防暴力破解

## 验证方式

1. 后端 `pytest tests/` 跑通
2. 手动测试：
   - 注册 → 返回 token + student
   - 登录 → 返回 token + student
   - 带 token 请求 → 正常返回
   - 不带 token / 错 token → 401
3. 前端 `npm run lint` 通过
4. 浏览器实测：注册 → 自动登录 → 跳主页 → 刷新保持登录
