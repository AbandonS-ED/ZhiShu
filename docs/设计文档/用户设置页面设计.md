# 用户设置页面 — 前端设计文档

> 版本: v1.0  |  日期: 2026-06-11  |  状态: ✅ **已落地**（2026-07-02 验收：3 后端端点 + setting 页 + Sidebar Zustand + 401 白名单修复 + 129 pytest 全过）

## 一、背景

`Header.tsx:63-68` 右上角有一个"设置"图标按钮（齿轮 SVG），但 **未绑定任何 onClick 事件**——点击不会跳转到任何页面，也不会打开弹窗。它是一个**死按钮**。

后端 `auth.py:1-169` 当前仅提供 `POST /auth/login`、`POST /auth/register`、`GET /auth/me/{student_id}` 三个端点，**没有任何修改个人信息或修改密码的接口**。

学生端没有任何"个人中心"或"账号设置"页面，用户无法修改姓名、邮箱、专业、年级，也无法修改密码。

## 二、目标

1. 激活 Header 右上角"设置"按钮 → 跳转 `/setting` 页面
2. 提供**个人信息编辑**（姓名、邮箱）+ **修改密码** + **账号只读展示**
3. 改完个人信息后，**Sidebar 底部用户信息实时刷新**（无刷页）
4. 修改后保持当前登录状态（JWT 不变）

## 三、非目标

不做：
- 主题切换（CSS 没有 dark mode 变量集）
- 头像上传（数据库无 avatar_url 字段）
- 通知偏好（项目无通知系统）
- 语言切换（纯中文项目）
- 专业/年级编辑（业务字段，应由管理员在后台改）

## 六、API 设计

### 6.1 新增后端端点

| 端点 | 方法 | 用途 | 鉴权 |
|------|------|------|------|
| `GET /api/v1/auth/me` | GET | 获取当前登录用户信息 | `get_current_user` |
| `PUT /api/v1/auth/me` | PUT | 更新姓名/邮箱 | `get_current_user` |
| `POST /api/v1/auth/change-password` | POST | 修改密码 | `get_current_user` |

**保留**现有的 `GET /auth/me/{student_id}` 端点（向后兼容，目前无前端调用，但 admin 后台将来可能用）。

### 6.2 Pydantic Schema

```python
class UpdateMeRequest(BaseModel):
    name: str = ""
    email: str = ""

class ChangePasswordRequest(BaseModel):
    old_password: str
    new_password: str

class StudentDTO(BaseModel):
    id: str
    student_no: str
    name: str = ""
    email: str = ""
    major: str = ""
    grade: str = ""
    role: str = "student"
    created_at: str = ""    # 新增：ISO 字符串
    last_login: str = ""    # 新增：ISO 字符串
```

### 6.3 业务规则

- **邮箱唯一性**：改邮箱时校验，不允许与他人重复
- **邮箱清空**：`""` → `None`（数据库允许多个 NULL，不允许多个 `""`）
- **新密码不能与旧密码相同**
- **改密码后保持登录**（JWT 不强制重登）

### 6.4 前端 API 客户端

```typescript
export const authApi = {
  // 现有
  login: (data) => request<AuthResponse>('/auth/login', { method: 'POST', body: ... }),
  register: (data) => request<AuthResponse>('/auth/register', { method: 'POST', body: ... }),

  // 新增
  getMe: () => request<AuthStudent>('/auth/me', { method: 'GET' }),
  updateMe: (data) => request<AuthStudent>('/auth/me', { method: 'PUT', body: JSON.stringify(data) }),
  changePassword: (data) => request<{ message: string }>('/auth/change-password', { method: 'POST', body: JSON.stringify(data) }),
}

interface AuthStudent {
  id: string
  student_no: string
  name?: string
  email?: string
  major?: string
  grade?: string        // 新增
  role?: string         // 新增（已有但前端未声明）
  created_at?: string   // 新增
  last_login?: string   // 新增
}
```

### 6.5 关键修正：`api.ts:27` 401 规则

**当前规则**：
```ts
if (res.status === 401 && !path.startsWith('/auth/')) { ... }
```
这把**所有** `/auth/*` 都排除，导致新的 `authApi.getMe()` 在 token 过期时**不会跳登录页**。

**修正**：白名单改为精确匹配
```ts
if (res.status === 401 && !path.startsWith('/auth/login') && !path.startsWith('/auth/register')) { ... }
```

## 七、状态管理

### 7.1 现有状态

- `Sidebar.tsx:89-101` 用 `useEffect + useState` 从 `localStorage.zhishu_student` 读取用户信息
- 依赖数组是 `[]`，**只在挂载时执行一次**
- **改完 localStorage 后 Sidebar 不会重新渲染**

### 7.2 修正方案

**用 Zustand store 响应式订阅**：

```typescript
// Sidebar.tsx
import { useAppStore } from '@/stores/appStore'

export default function Sidebar() {
  const student = useAppStore((s) => s.student)
  const setStudent = useAppStore((s) => s.setStudent)

  // 首次挂载时从 localStorage 同步到 store
  useEffect(() => {
    if (student) return
    const raw = localStorage.getItem('zhishu_student')
    if (raw) setStudent(JSON.parse(raw))
  }, [student, setStudent])

  // 直接从 store 派生（响应式）
  const userName = student?.name || student?.student_no || '未登录'
  // ...
}
```

**设置页保存成功后**：
```typescript
const updated = await authApi.updateMe({ name, email })
localStorage.setItem('zhishu_student', JSON.stringify(updated))
setStudent(updated)  // 触发所有订阅者 re-render，Sidebar 立即更新
```

`appStore.ts:35-66` 已有 `student` 状态和 `setStudent` action，**不需要修改 store**，只需要在 Sidebar/Header 中订阅。

## 九、错误处理

| 场景 | 提示 | 处理位置 |
|------|------|----------|
| 邮箱格式错误 | "邮箱格式不正确" | 前端正则 + 后端 Pydantic |
| 邮箱已被他人使用 | "该邮箱已被使用" | 后端唯一性校验 |
| 当前密码错误 | "当前密码不正确" | 后端 verify_password |
| 新密码 < 6 位 | "新密码至少 6 位" | 前端 + 后端 |
| 两次新密码不一致 | "两次输入的新密码不一致" | 前端校验 |
| 新密码 = 旧密码 | "新密码不能与旧密码相同" | 后端 |
| 401 token 过期 | 自动跳登录页 | `api.ts:27`（修正后） |
| 网络错误 | "保存失败" | catch 兜底 |

**错误展示**：使用 `<div className="set-msg err">` 红色提示框，**不用 alert**。

## 十二、风险点

| # | 风险 | 缓解 |
|---|------|------|
| 1 | 邮箱 unique 约束 + `""` 冲突 | 写库前 `.strip() or None` 归一 |
