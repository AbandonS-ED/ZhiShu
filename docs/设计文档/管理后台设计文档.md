> 版本：v1.0 | 日期：2026-06-11 | 状态：✅ **已落地**（2026-07-04 验收：9 页面 1:1 复刻 houtai.html + 批量删除 6 页 + 题库 CRUD 6 端点 + JWT 角色隔离 + zhishu_admin_token + 67 端点中 18 个管理端 API 全部实联）

## 一、项目概述

### 1.1 背景
智枢(SmartHub)系统需要一个管理后台，用于管理用户、审核内容、监控系统运行状态。

### 1.2 目标
- 提供管理员对系统数据的可视化管理能力
- 支持用户管理、内容审核、数据统计
- 与现有学生端系统共用后端，独立前端路由

### 1.3 技术选型
| 层 | 选型 | 说明 |
|---|---|---|
| 前端 | 复用 Next.js 14 | `/admin` 路由前缀 |
| 样式 | 复用 Tailwind + globals.css | 保持风格一致 |
| 后端 | 复用 FastAPI | 新增 `/api/v1/admin` 路由 |
| 认证 | 复用 JWT | role 字段区分权限 |

### 2.1 功能清单

| 模块 | 功能 | 优先级 | 说明 |
|------|------|--------|------|

| **用户管理** | 删除用户 | P2 | 硬删除，级联清理 |
| **资源管理** | 资源列表 | P0 | 分页/按用户筛选 |
| **资源管理** | 查看详情 | P0 | 查看完整内容 |
| **资源管理** | 删除资源 | P1 | 硬删除 |
| **练习题** | 题目列表 | P0 | 分页/按用户筛选 |
| **练习题** | 删除题目 | P1 | 硬删除 |

### 2.2 用户角色

| 角色 | 权限 |
|------|------|
| `admin` | 全部管理功能 |
| `student` | 仅学生端功能，禁止访问 /admin/* |

**查询逻辑：**
```python
# 用户总数
SELECT COUNT(*) FROM students WHERE role = 'student';

# 资源总数
SELECT COUNT(*) FROM resources;

# 今日活跃（有学习记录的用户）
SELECT COUNT(DISTINCT student_id) 
FROM learning_records 
WHERE created_at >= CURRENT_DATE;

# 今日生成资源
SELECT COUNT(*) 
FROM resources 
WHERE created_at >= CURRENT_DATE;
```

## 六、权限控制

### 6.1 后端权限

```python
# core/dependencies.py 新增

async def get_current_admin(user: Student = Depends(get_current_user)):
    """验证当前用户是管理员"""
    if user.role != "admin":
        raise HTTPException(status_code=403, detail="需要管理员权限")
    return user
```

### 6.2 路由保护

```python
# admin 路由全部使用 get_current_admin
from app.core.dependencies import get_current_admin

@router.get("/stats")
async def get_stats(admin: Student = Depends(get_current_admin)):
    ...
```

### 6.3 前端权限

```typescript
// admin 布局文件 admin/layout.tsx
'use client'

import { useEffect, useState } from 'react'
import { useRouter } from 'next/navigation'

export default function AdminLayout({ children }) {
  const router = useRouter()
  const [isAuth, setIsAuth] = useState(false)

  useEffect(() => {
    const token = localStorage.getItem('zhishu_admin_token')
    const user = JSON.parse(localStorage.getItem('zhishu_admin_user') || '{}')
    
    if (!token || user.role !== 'admin') {
      router.push('/admin/login')
    } else {
      setIsAuth(true)
    }
  }, [])

  if (!isAuth) return null
  
  return (
    <div className="admin-layout">
      <Sidebar />
      <main>{children}</main>
    </div>
  )
}
```

### 6.4 Token 隔离

```
学生端 token: zhishu_token
管理端 token: zhishu_admin_token
```

避免管理员在学生端误操作。

## 九、注意事项

### 9.1 安全
- 管理员密码使用 bcrypt 哈希
- JWT 密钥与学生端共用，但 role 字段必须校验
- 删除操作需要二次确认
- 不暴露敏感信息（密码哈希、API Key）

### 9.2 性能
- 列表接口必须分页，禁止全量查询
- 搜索使用数据库 LIKE，不使用全文索引（数据量小）
- 趋势数据可缓存 5 分钟

### 9.3 兼容性
- 现有 `/auth/login` 不变，管理员走 `/admin/login`
- 学生端代码零改动
- 数据库只加字段，不改现有结构
