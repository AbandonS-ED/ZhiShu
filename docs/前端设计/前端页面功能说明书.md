## 2. 页面一：登录/注册页

### 2.1 登录页 (`/login`)

- 邮箱输入框
- 密码输入框
- 登录按钮（loading 状态）
- "没有账号？去注册" 链接跳转 `/register`
- 表单验证（邮箱格式、密码非空）
- 登录成功 → 跳转首页

### 2.2 注册页 (`/register`)

- 姓名输入框
- 邮箱输入框
- 密码输入框
- 确认密码输入框
- 专业输入框（下拉选择或自由输入）
- 年级输入框（大一/大二/大三/大四/研一/研二/研三）
- 注册按钮（loading 状态）
- "已有账号？去登录" 链接跳转 `/login`
- 注册成功后自动登录并跳转首页

### 2.3 后端 API 对应

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/auth/login` | 登录 |
| POST | `/api/v1/auth/register` | 注册 |
| GET | `/api/v1/auth/profile` | 获取当前用户信息 |

### 4.3 功能点清单

#### 4.3.1 消息列表区域

- 用户消息和 AI 消息按时间顺序排列
- 消息气泡：用户消息右对齐（蓝色），AI 回复左对齐（灰色/白色）
- 流式渲染 AI 回复（SSE 驱动，打字机效果）
- 消息内容支持以下渲染类型：
  - **普通文本**：Markdown 渲染（标题、列表、表格、加粗、链接等）
  - **代码块**：语法高亮（react-syntax-highlighter），带复制按钮、语言标签
  - **数学公式**：（如有需要，可接入 KaTeX/MathJax）
  - **思维导图**：Mermaid 代码块 → 渲染为可视化图表
  - **资源卡片**：当 AI 返回 resource 类型数据时，渲染为内嵌卡片
  - **进度条**：显示多 Agent 生成进度
  - **引用标注**：`[1]` 可点击跳转引用源

#### 4.3.2 资源生成进度区域

当用户触发"资源生成"时，显示并排的 Agent 进度条：

| Agent | 状态 | 颜色 |
|-------|------|------|
| 文档 Agent | 待处理/生成中/已完成/失败 | 灰色/蓝色/绿色/红色 |
| 思维导图 Agent | 同上 | 同上 |
| 出题 Agent | 同上 | 同上 |
| 代码 Agent | 同上 | 同上 |
| 视频 Agent | 同上 | 同上 |

- 进度：显示百分比或步骤描述（如"正在检索知识库..."）
- 实时更新（通过 SSE / WebSocket）

#### 4.3.3 输入框区域

- 多行文本输入框（`textarea`），支持 Enter 发送，Shift+Enter 换行
- 发送按钮（带 loading 状态）
- 输入框自动调整高度
- 发送后清空输入框，焦点的保留
- 禁用状态：回复未完成时禁用发送

#### 4.3.4 会话管理

- 新建对话按钮
- 会话历史列表（可选，侧边栏或下拉菜单）
- 每个会话保留上下文

### 4.4 流式输出（SSE）协议

```typescript
// 前端接收的 SSE event 格式
interface SSEEvent {
  type: 'progress' | 'result' | 'message' | 'error' | 'done';
  agent?: string;           // 哪个 Agent
  progress?: number;        // 进度 0-1
  step?: string;            // 当前步骤描述
  data?: {
    // message 类型
    content?: string;
    // result 类型
    title?: string;
    content_text?: string;  // Markdown 内容
    mermaid_code?: string;  // 思维导图 Mermaid 代码
    code_snippets?: Array<{language: string, code: string, explanation?: string}>;
    exercises?: Array<Exercise>;
    audio_url?: string;
    // 公共
    source_refs?: Array<{doc_id: string, chunk_id: string, page: number, quote: string}>;
  };
}
```

### 4.5 辅助功能

- 消息复制按钮（hover 出现）
- 消息点赞/点踩反馈（hover 出现）
- 重新生成按钮（hover 出现）
- 消息滚动到底部（自动 + 手动按钮）
- 输入框防抖（避免频繁发送）

### 4.6 后端 API 对应

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/chat/stream` | 流式对话 SSE |
| POST | `/api/v1/profile/build` | 画像构建（对话式） |
| POST | `/api/v1/resource/generate` | 生成资源（流式） |
| POST | `/api/v1/tutor/ask` | 智能辅导问答 |
| POST | `/api/v1/path/generate` | 生成学习路径 |
| WS | `/ws/{client_id}` | WebSocket 连接（进度推送） |

### 5.2 功能点清单

- **六维雷达图**：使用 Chart.js 或 D3.js 绘制
  - 6 个维度：知识基础、认知风格、学习目标、易错点、学习节奏、兴趣方向
  - 可交互（hover 显示详情）
- **画像完整性**：进度环或进度条，显示完整度百分比
- **对话完善画像**：点击跳转 `/chat` 并自动发起画像对话
- **画像版本信息**：当前版本号、最近更新时间
- **知识点掌握度列表**：
  - 列表形式，每项显示：知识点名称 + 进度条 + 掌握百分比
  - 按掌握度排序（低到高）
  - 颜色编码：< 30% 红色、30-70% 黄色、>70% 绿色
- **薄弱环节标签**：Tag 组件显示薄弱知识点
- **画像详情展开**：点击每个维度查看详细 JSON 数据

## 14. 注意事项

### 14.1 Next.js App Router 注意点

- Mermaid 和 ReactFlow 组件必须使用 `dynamic()` 导入并设置 `{ ssr: false }`
- 使用 `'use client'` 指令标记客户端组件
- 全局布局在 `app/layout.tsx` 中定义

### 14.2 SSE 流式渲染注意点

- SSE 使用 `fetch` + `ReadableStream` 而非 `EventSource`（POST 请求需要）
- 流式文本需要管理缓冲区和增量渲染
- 打字机效果 = 逐 chunk 追加到 DOM

### 14.3 数据一致性

- 画像更新后 → 导航栏/首页相关数据同步刷新
- 资源生成后 → 资源中心列表自动更新
- 路径更新后 → 路径页面重新渲染

### 14.4 开发优先级（前端）

| 优先级 | 页面/功能 | 理由 |
|--------|-----------|------|
| P0 | 全局布局 + 路由 | 基础骨架 |
| P0 | 登录/注册 | 用户入口 |
| P0 | 主对话窗口（流式 + Markdown + 卡片） | F1/F2/F3 核心交互入口 |
| P0 | 学习画像页（雷达图） | F1 展示 |
| P0 | 资源中心（列表 + 卡片 + 详情） | F2 展示 |
| P0 | 学习路径页（ReactFlow） | F3 展示 |
| P1 | 智能辅导页 | F4 加分 |
| P1 | 学习评估/统计页 | F5 加分 |
| P1 | 练习题交互组件 | 核心交互 |
| P2 | Agent 进度追踪 | 非功能性 |
