## 1. 快速开始

### 1.1 访问系统

**演示部署**（本机开发）：

| 服务 | 地址 | 启动命令 |
|---|---|---|
| 前端 | http://localhost:3000 | `cd frontend && npm run dev` |
| 后端 | http://localhost:8001 | `cd backend && venv/Scripts/python -m uvicorn app.main:app --host 0.0.0.0 --port 8001` |
| **学生端入口** | **http://localhost:3000** | 注册 → 登录 → 15 页面 |
| **管理端入口** | **http://localhost:3000/admin/login** | admin / admin123 → 9 页面 |
| API 文档 | http://localhost:8001/docs | Swagger 自动生成 |

### 1.2 启动顺序

```bash
# 1. 启动 PostgreSQL（本地安装）
# 2. 初始化数据库 + 管理员账号
psql -U postgres -f backend/scripts/init_db.sql
cd backend && venv\Scripts\python scripts\init_admin.py

# 3. 启动后端（终端 1）
cd backend && venv\Scripts\python -m uvicorn app.main:app --host 0.0.0.0 --port 8001

# 4. 启动前端（终端 2）
cd frontend && npm run dev
```

启动后浏览器打开：
- **学生端**: `http://localhost:3000`（首次需要注册）
- **管理端**: `http://localhost:3000/admin/login`（admin / admin123）
- 快速开始按钮（直接进入各模块）
- 课程进度条

> student_id 在第一次访问时由前端 `lib/student.ts` 生成 UUID 写入 localStorage，后续请求都带这个 ID。

## 2. 7 个功能页面

### 2.1 `/duihua` — 智能对话（核心入口）

**功能**：自然语言对话触发多智能体，自动路由到最合适的 Agent。

**使用步骤**：
1. 页面打开即可看到历史会话标签（可新建）
2. 在底部输入框输入问题
3. 按 Enter 发送，看到 AI **逐字流式**返回
4. 右侧侧栏显示已生成资源、推荐问题、当前画像的知识点

**支持的请求类型**（`_quick_route` 关键词路由 + `master_agent.route` 兜底）：

| 输入示例 | 路由 | 实际效果 |
|---|---|---|
| "讲解 A* 搜索算法的原理" | tutor | 真逐 token 流式讲解 |
| "什么是梯度下降？" | tutor | 真逐 token 流式解释 |
| "出 5 道搜索算法练习题" | exercise | ⭐ **dual-format 真流式**渲染 5 道题 |
| "画一下机器学习的思维导图" | mindmap | 触发 Mermaid 思维导图生成 |
| "生成神经网络的学习路径" | path | 触发路径规划 |
| "分析我的学习情况" | profile | 触发画像分析 |

**⭐ dual-format 流式说明**（重要体验特性）：输 "出 5 道搜索算法练习题" 时，**题目会逐行实时渲染**（不是等几秒再返回整段 JSON），是 2026-06-09 修复的关键体验 bug。

### 2.2 `/profile` — 学习画像

**功能**：查看 AI 提取的 6 维学习画像 + 触发对话式画像更新。

**页面元素**：
- 6 维雷达图（知识掌握 / 学习风格 / 认知水平 / 兴趣 / 薄弱点 / 学习节奏）
- 知识点掌握度列表（带进度条）
- 薄弱环节标签
- 6 维详情（可展开）
- "🤖 AI 对话式画像" 按钮 → 弹出 `ChatModal.tsx` 弹窗

**使用步骤**：
1. 打开 `/profile` 看当前画像
2. 点 "🤖 AI 对话式画像" → 弹窗打开
3. 输 "我是计算机大三学生，想考研" → AI 追问
4. 回答 3-5 轮 → 看到新版本画像（`version` 递增，`is_current=true`）

### 2.3 `/resources` — 资源中心

**功能**：展示 / 生成学习资源（文档、思维导图、练习题、代码、音频）。

**页面元素**：
- 顶部筛选栏（全部 / 文档 / 思维导图 / 练习题 / 代码 / 音频）
- 搜索框
- 资源卡片瀑布流（带收藏、详情模态框）
- 顶部输入框 → 点 "生成" 触发 SSE 流式

**使用步骤**：
1. 顶部输入 "神经网络基础"
2. 点 "生成" → 看到 AI 实时流式生成内容
3. 关闭生成弹窗，资源已加入右侧"已生成资源"

### 2.4 `/tiku` — 练习题库

**功能**：刷练习题 + AI 出题。

**使用步骤**：
1. 顶部输入 "搜索算法" → 点 "出 5 题" → AI 流式生成 5 道题
2. 答题：选择 / 判断 / 简答 / 编程
3. 即时反馈（看答案 + 解析）
4. 下方进度环 + 知识点分析 + 最近答题

### 2.5 `/path` — 学习路径

**功能**：DAG 路径可视化 + 个性化学习计划。

**使用步骤**：
1. 顶部输入 "机器学习, 深度学习, CNN"（逗号分隔）
2. 点 "生成 14 天路径" → AI 生成 DAG 节点 + 边 + 每日计划
3. 点击节点查看详情面板（前置知识、关联资源、状态）
4. 下方 5 天每日计划

### 2.6 `/pinggu` — 学习评估

**功能**：学习效果评估 + 评分展示 + AI 智能问答。

**页面元素**：
- 评分环形动画（基于 `evaluation_service.py`）
- 6 维进度条（knowledge_mastery 等）
- 趋势折线图（学习时长）
- 正确率柱状图（按知识点）
- AI 评估入口（`/api/v1/tutor/ask`）
- 学习记录分页

**使用步骤**：
1. 打开页面看历史评估数据
2. 顶部输入问题 → 点 "提交" → AI 智能评估回复

### 2.7 `/` — 仪表盘

聚合 4 个统计卡 + 最近活动 + 课程进度，是系统入口。

## 3. 关键技术体验

### 3.1 ⭐ 真流式输出（最直观体验）

**对比测试**：
1. 打开 `/duihua`
2. 输 "出 5 道搜索算法练习题"
3. **观察**：题目是从上到下**逐字**出现的，不是先等几秒再一次性出现
4. 题目出现完后，右侧"已生成资源"会新增 "搜索算法 · 练习题 · 完成"

**底层原理**：LLM 一份输出 = 人类可读 markdown + `---JSON_DATA---` 分隔 + 结构化 JSON；后端 chat.py 实时 emit token，遇分隔符停止（详见 `CLAUDE.md` "练习题 dual-format 流式协议"）

### 3.2 RAG 增强智能辅导

输 "什么是梯度下降？" 触发 tutor 路由：
- 检索 `document_chunks` 表（pgvector / JSONB 降级）
- 找到最相关的 3-5 个文档块
- LLM 基于这些块生成回答
- 添加引用源标注

### 3.3 防幻觉三层验证

`/duihua` 输技术问题，Document Agent / Exercise Agent 输出时会经过 `anti_hallucination.py`：
- PatternDetector 正则检测
- SourceValidator 引用验证
- LLMValidator 语义校验

## 4. 常见问题

### Q1: student_id 在哪里？怎么多人用？

A: `student_id` 由前端 `lib/student.ts` 在 localStorage 生成 UUID。**多人在同一浏览器看到的是同一画像**——本项目作为 demo，未实现用户系统。生产环境需加登录。

### Q2: 后端连不上？

A: 检查端口（默认 8001，匹配 `frontend/src/lib/api.ts:5` 的 `BASE_URL`）。`8000` 在 Windows 上有"僵尸 socket"问题。

### Q3: 流式输出看不到逐字效果？

A: 后端必须用新代码（chat.py 包含 dual-format 实现）。检查 `git log backend/app/api/chat.py` 确认有 2026-06-09 的修复。

### Q4: 中文乱码？

A: 浏览器设 UTF-8。后端 .env 配 `PYTHONIOENCODING=utf-8`。

### Q5: 画像数据怎么清？

A: 浏览器 DevTools → Application → Local Storage → 删 `student_id` 项。

### Q6: 如何切换讯飞星火 V4？

A: `backend/.env` 配 `LLM_PROVIDER=spark` + `SPARK_API_KEY=xxx` + `SPARK_APP_ID=xxx`，重启后端即可（lifespan 自动选客户端）。注意：当前测试用的是小米 MiMo v2.5（开发），上线前需切到讯飞星火 V4。
