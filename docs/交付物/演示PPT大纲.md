### 第6页：核心功能2 — 多智能体资源生成（60秒，重点演示）

**标题**：F2: 多资源类型生成，⭐ 真流式输出

**演示要点**：
1. 打开 `/duihua` 页面
2. **演示 1**（普通对话）：输 "讲解 A* 搜索算法的原理" → 看到 tutor agent 真逐 token 流式返回
3. **演示 2**（⭐ **dual-format 流式**）：输 "出 5 道搜索算法练习题" → 看到题目**逐行实时渲染**（不是等几秒再返回的伪流式）
4. **演示 3**（思维导图）：输 "画一下 A* 搜索的思维导图" → 触发 mindmap 路由 → 看到 Mermaid 渲染

**资源类型**：
| 类型 | Agent | 输出 |
|------|-------|------|
| 学习文档 | Document Agent | Markdown 文档 |
| 思维导图 | MindMap Agent | Mermaid 图 |
| 练习题 | Exercise Agent | 多类型题目（⭐ dual-format 真流式） |
| 学习路径 | Path Agent | DAG + 每日计划 |
| 智能辅导 | Tutor Agent | RAG 问答 |

**⭐ 真流式技术亮点**：
- 普通 LLM 直接吐 JSON 用户体验差（满屏 `{"exercises":[...]}`）
- **dual-format 协议**：LLM 一份输出 = 人类可读 markdown + `---JSON_DATA---` 分隔 + 结构化 JSON
- 后端 chat.py 边收 token 边推给前端，遇 `---JSON_DATA---` 停止 emit
- 前端 duihua/page.tsx 用 `markdownToHtml()` 实时渲染

### 第7页：核心功能3 — 个性化路径（50秒）

**标题**：F3: 拓扑排序，个性化学习路径

**演示要点**：
1. 打开 `/path` 页面，输 "机器学习, 深度学习, CNN"
2. 点 "生成 14 天路径" → 看到 DAG 路径生成
3. 点击节点查看详情面板

**算法思路**：
```
1. 获取课程知识图谱
2. 过滤已掌握知识点（基于画像 knowledge_mastery）
3. 拓扑排序（Kahn 算法，确保先修在前）
4. 薄弱知识点优先（基于 weak_topics）
5. 根据 daily_hours 分配每日计划
```

**创新点**：
- 基于画像的个性化调整
- 动态更新（学习后重新规划）

### 第10页：防幻觉机制（40秒）

**标题**：RAG + 三层验证 + 引用源，可信内容生成

**机制说明**（`backend/app/services/anti_hallucination.py`）：
```
1. 检索阶段：只使用知识库内容（pgvector + JSONB fallback）
2. 生成阶段：Prompt 约束"只基于上下文回答"
3. 验证阶段：
   a. PatternDetector — 正则检测虚假年份/期刊/统计数据/绝对化表述
   b. SourceValidator — 验证引用格式是否有来源支撑
   c. LLMValidator — LLM 语义一致性校验
4. 标注阶段：添加引用源 [1][2]
```

**效果展示**：
- 展示有引用源的回答
- 展示 `hallucination_risk` 分数

## FAQ 预案

| 评委可能问 | 答 |
|---|---|
| 为什么不用 LangGraph？ | 7 个 Agent + Master 路由用直接调用 LLM 已足够，StateGraph 复杂度对项目规模过度设计 |
| 流式真的是逐 token 吗？ | 是。tutor/chat 是真逐 token；⭐ 2026-06-09 新增 dual-format 协议，练习题也升级为真流式 |
| 讯飞星火 V4 怎么切换？ | `.env` 配 `LLM_PROVIDER=spark` + `SPARK_API_KEY`，lifespan 自动切换客户端 |
| 防幻觉如何验证？ | 三个层面：模式正则、引用验证、LLM 语义校验。详见 anti_hallucination.py |
| pgvector 没装怎么办？ | 走 JSONB fallback 方案，余弦相似度用 Python numpy 计算 |
| 端口冲突怎么解决？ | 后端用 8001 绕开 Windows 8000 僵尸 socket；前端 `api.ts:5` 同步配置 |
