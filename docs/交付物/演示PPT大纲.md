### 第2页：问题与痛点（30秒）

**标题**：高等教育的三大痛点

| 痛点 | 描述 | 数据 |
|------|------|------|
| 学生差异大 | 知识基础、学习能力、兴趣方向各不相同 | 70%学生认为课程不适合自己 |
| 资源繁杂无序 | 学习资源数量庞大但缺乏个性化组织 | 平均每门课500+资源 |
| 教学模式单一 | 课堂集体讲授无法照顾个体节奏 | 85%课堂仍是传统讲授 |

**关键信息**：需要"因材施教"的数字化解决方案

### 第4页：系统架构（40秒）

**标题**：多智能体协同架构

```
用户 → API Gateway → Master Agent
                         │
         ┌───────┬───────┼───────┬───────┐
         ▼       ▼       ▼       ▼       ▼
      画像    文档     导图    出题    代码
      Agent   Agent   Agent   Agent   Agent
         │       │       │       │       │
         └───────┴───────┴───────┴───────┘
                         │
                    RAG知识库
                  (pgvector)
```

**技术亮点**：
- LangGraph状态图管理Agent协同
- 条件路由 + 并行执行
- RAG检索增强防幻觉

---

### 第5页：核心功能1 — 对话式画像（50秒）

**标题**：F1: 六维学习画像，对话式构建

**演示要点**：
1. 展示对话界面
2. 用户说"我是计算机大三学生，想考研"
3. 系统自动提取6维度特征
4. 展示生成的画像

**6个维度**：
```
知识基础 | 认知风格 | 学习目标 | 易错点 | 学习节奏 | 兴趣方向
```

**创新点**：
- 非表单式，自然对话构建
- 随学随新，画像持续更新

---

### 第6页：核心功能2 — 多智能体资源生成（60秒）

**标题**：F2: 5种资源，多Agent并行生成

**演示要点**：
1. 请求"生成神经网络学习资料"
2. 展示多Agent并行执行过程
3. 展示生成的5种资源

**5种资源类型**：
| 类型 | Agent | 输出 |
|------|-------|------|
| 学习文档 | Document Agent | Markdown文档 |
| 代码示例 | Document Agent | Python代码 |
| 音频脚本 | Document Agent | 脚本文本 |
| 思维导图 | MindMap Agent | Mermaid图 |
| 练习题 | Exercise Agent | 多类型题目 |

**技术亮点**：
- 每个Agent有独立Prompt+工具+输出规范
- Master Agent统一调度，支持并行
- 基于RAG确保内容准确

---

### 第7页：核心功能3 — 个性化路径（50秒）

**标题**：F3: 拓扑排序，个性化学习路径

**演示要点**：
1. 展示学习路径DAG图
2. 说明路径如何个性化
3. 展示每日学习计划

**算法思路**：
```
1. 获取课程知识图谱
2. 过滤已掌握知识点
3. 拓扑排序（确保先修在前）
4. 薄弱知识点优先
5. 根据每日时长分配
```

**创新点**：
- 基于画像的个性化调整
- 动态更新（学习后重新规划）

---

### 第8页：核心功能4 — 智能辅导（40秒）

**标题**：F4: RAG增强的智能辅导

**演示要点**：
1. 学生提问"什么是梯度下降？"
2. 系统检索知识库
3. 生成解答并标注引用源

**技术亮点**：
- RAG检索确保答案有据可查
- 引用源标注增加可信度
- 防幻觉机制验证答案准确性

---

### 第9页：核心功能5 — 效果评估（30秒）

**标题**：F5: 学习行为追踪与评估

**演示要点**：
1. 展示学习仪表盘
2. 展示练习正确率统计
3. 说明行为如何触发画像更新

**数据展示**：
- 学习时长统计
- 知识点掌握度
- 资源使用情况
- 练习正确率

---

### 第10页：防幻觉机制（40秒）

**标题**：RAG + 引用源，可信内容生成

**机制说明**：
```
1. 检索阶段：只使用知识库内容
2. 生成阶段：Prompt约束基于上下文
3. 验证阶段：检测无依据陈述
4. 标注阶段：添加引用源 [1][2]
```

**效果展示**：
- 展示有引用源的回答
- 展示"知识库中无此内容"的拒绝回答

---

### 第11页：技术栈（30秒）

**标题**：技术选型

| 层级 | 技术 |
|------|------|
| 前端 | Next.js 14 + shadcn/ui + ReactFlow |
| 后端 | FastAPI + LangGraph |
| 大模型 | 讯飞星火V4 |
| 向量库 | PostgreSQL + pgvector |
| 实时通信 | SSE + WebSocket |

**选型理由**：
- LangGraph：状态管理强，支持并行执行
- pgvector：与业务数据统一存储
- 讯飞星火：满足赛题硬约束
