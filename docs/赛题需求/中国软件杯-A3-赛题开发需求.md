## 1. 项目目标

构建一个**多智能体协同系统**，基于大模型技术为高等教育学生**自动生成个性化、多模态学习资源**，实现"因材施教"的数字化落地。

**核心问题**：
- 学生：知识基础、学习能力、兴趣方向差异大，标准化教学匹配差
- 资源：学习资源繁杂无序，难以精准匹配
- 模式：课堂集体讲授无法照顾每个学生的学习节奏

**系统定位**：以某一具体专业课程（如人工智能、计算机、电子信息等）为切入点，开发一个**学习资源智能生成 + 路径规划 + 多模态辅导**的平台。

## 2. 核心功能需求（共 5 项，2 项为加分项）

### F1. 对话式学习画像自主构建【必做 · 35% 创新分】

**目标**：通过自然语言对话（而不是传统表单）自动抽取学生特征，构建动态学习画像。

**关键要求**：
- **≥ 6 个维度**，例如：
  - 知识基础（如：先修知识掌握度、当前水平）
  - 认知风格（视觉型/听觉型/动手型）
  - 学习目标（如：考研/工作/竞赛）
  - 易错点偏好
  - 学习节奏（每日可投入时长）
  - 兴趣方向 / 偏好的资源类型
- **支持对话式抽取**：用户用自然语言描述，系统自动抽取结构化特征
- **随学随新**：画像随学习行为持续更新（不是一次性建好就锁死）
- **存储**：画像以结构化形式存储，可被下游智能体读取

**输入**：学生专业、学习目标、学习历史、对话内容
**输出**：结构化学生画像（JSON / 数据库表）
**交互方式**：对话窗口，多轮上下文

---

### F2. 多智能体协同的资源生成【必做 · 45% 技术分 · 核心】

**目标**：由不同角色的智能体协作，**至少生成 5 种类型**的个性化学习资源。

**关键要求**：
- **必须体现"多智能体架构"** —— 不能是一个大模型硬撑，要有清晰的角色分工
- **≥ 5 种资源类型**（建议从以下选）：

| # | 资源类型 | 智能体角色示例 | 输出形式 |
|---|---|---|---|
| 1 | 课程讲解文档 | 教研员 Agent | Markdown / PDF |
| 2 | 知识点思维导图 | 知识图谱 Agent | Mermaid / XMind / 图片 |
| 3 | 练习题目（多种类型） | 出题 Agent | JSON / 卡片 |
| 4 | 拓展阅读材料 | 资源推荐 Agent | 文章链接 / 摘要 |
| 5 | 多模态教学视频/动画 | 视频生成 Agent | MP4（流式） |
| 6 | 代码类实操案例 | 编程辅导 Agent | 代码文件 + 讲解 |

**架构设计要点**：
- 智能体之间**能通信 / 任务分发**（Master-Worker 或 Peer-to-Peer）
- 中央调度 Agent 接收请求 → 拆解任务 → 分发给专项 Agent → 汇总结果
- 每个 Agent 应有**独立的 Prompt + 工具 + 输出规范**

**输入**：学生画像 + 课程知识库 + 学习需求
**输出**：至少 5 种个性化资源

---

### F3. 个性化学习路径规划 + 资源推送【必做】

**目标**：根据学生画像 + 资源库，**动态生成学习路径**（步骤+顺序），并精准推送资源。

**关键要求**：
- **路径生成**：结合专业、学习进度、知识掌握度，输出结构化学习计划（章节顺序+预计时长）
- **动态调整**：根据学习效果数据/对话反馈，更新路径
- **精准推送**：基于画像的标签匹配，推送对应类型的资源（文档/视频/题库/代码）
- **依赖 F1 + F2**：路径规划需要画像，资源来自 F2

**输入**：学生画像 + 已有资源池
**输出**：学习路径（JSON/可视化）+ 推荐资源列表

---

### F4. 智能辅导【加分项 · 建议做】

**目标**：学生遇到问题时，提供**即时、多模态答疑**。

**关键要求**：
- 多模态解答：文字 + 图解 + 短视频讲解
- 上下文感知：知道学生在学什么、卡在哪里
- RAG：基于课程知识库回答，避免幻觉

---

### F5. 学习效果评估【加分项 · 建议做】

**目标**：跟踪学习行为 + 评估效果 + 动态调整推送策略。

**关键要求**：
- 行为跟踪：哪些资源看了 / 做了 / 收藏了
- 练习评分：题库正确率统计
- 反馈循环：评估结果回流到画像 + 路径规划

---

## 3. 非功能性需求（必做，会影响"功能实现"45% 评分）

### N1. 界面与交互
- 美观大方、符合现代 AI 产品规范
- **流式输出**（SSE / WebSocket）
- **Markdown 渲染**
- **多模态内容卡片化展示**
- 无明显功能 / 界面错误

### N2. 开源与合规
- 用了哪些开源项目 / 工具 → **提交文档显著位置标注**（名称、来源、协议）
- 严格遵循开源协议

### N3. 防幻觉 + 内容安全【极其重要 · 评分重点】
- 学术内容**无事实性错误**（这点 RAG 强相关）
- 无敏感违规信息
- 建议用 RAG（检索增强生成）+ 引用源标注

### N4. 性能
- 智能体核心功能响应时间合理
- 多模态资源生成 → 必须有**生成进度追踪**或**流式呈现**（不能白屏等几秒）
- 推荐：长任务用后台任务队列 + 进度回调

### N5. 必用工具约束
- **开发过程中使用的其他 AI 辅助工具，需选用科大讯飞相关工具** ⚠️
- 多智能体框架**不限**（LangChain / AutoGen / CrewAI / 自研均可）
- 编程语言/数据库/编辑器/硬件**均不限制**

## 4. 技术架构建议（供 Claude Code 参考）

### 4.1 推荐技术栈

| 层 | 选型 | 理由 |
|---|---|---|
| **前端** | Next.js 14 (App Router) + TailwindCSS + shadcn/ui | 流式渲染支持好、AI 卡片化组件丰富 |
| **后端** | FastAPI (Python) / Express (Node.js) | 都可，Python 对 AI 生态更友好 |
| **多智能体框架** | **LangGraph**（推荐）/ AutoGen / CrewAI | LangGraph 状态管理强、生产可用 |
| **大模型** | 讯飞星火 V4（必选）+ OpenAI/Claude 作辅助 | 满足"用讯飞工具"硬约束 |
| **向量库** | Chroma / Milvus / pgvector | 课程知识库 RAG |
| **关系库** | PostgreSQL | 学生画像、学习记录 |
| **任务队列** | Celery / Redis Streams | 多模态生成后台任务 |
| **对象存储** | MinIO / 阿里云 OSS | 生成的视频/图片存储 |

### 4.3 多智能体协同流程（Master-Worker 模式）

```
1. 用户进入系统 → 调度 Agent 启动
2. 调度 Agent 调用画像 Agent：构建/更新学生画像
3. 调度 Agent 根据画像 + 学习目标 → 拆解为子任务
4. 调度 Agent 派发给各专项 Agent（可并行）
   - 文档 Agent → 生成讲解文档
   - 思维导图 Agent → 生成思维导图
   - 出题 Agent → 生成练习题
   - 代码 Agent → 生成代码案例
   - 视频 Agent → 生成讲解视频
5. 调度 Agent 收集结果 → 调路径规划 Agent 生成学习路径
6. 调度 Agent 流式返回给前端
7. 用户行为数据回流 → 更新画像 + 优化推送
```

## 10. 关键避坑点

1. **"多智能体"不能是幌子** —— 评审会看架构和代码，Agent 之间要有**真实的协同和通信**，不是几个 if-else 调不同 Prompt
2. **"至少 5 种资源"必须实打实** —— 不能只列 5 种但只做了 2 种
3. **"防幻觉"必须有 RAG** —— 不能让大模型自由发挥，引用源标注是亮点
4. **"流式输出"必须真流** —— 不能先等几秒再返回结果
5. **必须用讯飞工具** —— 这是硬约束，**不要全用 OpenAI/Claude**
6. **文档要详细** —— 评委不一定有时间跑你的代码，文档和 PPT 是第一印象
7. **演示视频 7 分钟** —— 时间卡死，重点展示核心功能 + 多模态效果

## 11. Claude Code 开发建议

1. **先跑通最小闭环**：
   - 1 门课程 + 1 个学生 + Master Agent + 1 个子 Agent（文档 Agent）
   - 验证流式输出 + RAG 后再扩展
2. **分阶段提交**：
   - 每个 Phase 完成后跑一次演示，确认效果
3. **善用 Claude Code 自身**：
   - 本题赛题示例明确提到"AI 辅助编程工具 (如 Claude Code 等)"
   - 用 Claude Code 本身参赛完全合规（但要在文档里声明）
4. **多智能体优先选 LangGraph**：
   - 状态管理 + 持久化 + 流式输出 = 完美契合本题
   - 比纯 Prompt 串接"专业"
5. **视频生成如果成本高**：
   - 可以用**音频 + 幻灯片**的方式代替真视频
   - 或用 AI 数字人 API（讯飞有声 / 开源 SadTalker）
   - 关键是要"多模态"，不一定要真拍视频
