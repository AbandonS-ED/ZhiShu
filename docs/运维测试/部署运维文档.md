```
                    └──────┬──────┘
```

## 3. 快速部署（本地裸跑）

### 3.1 克隆代码

```bash
git clone https://github.com/AbandonS-ED/ZhiShu.git
cd ZhiShu
```

### 3.2 初始化数据库（首次）

```bash
# 进入 PostgreSQL
psql -U postgres -f backend/scripts/init_db.sql
# 输入密码：123456
# 该脚本会建库 zhishu + 9 张表（包含 admin 账号种子数据）
```

### 3.2.1 初始化管理员账号（2026-06-11 新增）

> ⚠️ **PowerShell `$2b$` 变量插值坑**：在 PowerShell 命令行直接传 `'$2b$12$...'` 会被 shell 解析为变量，破坏 bcrypt 哈希。**必须用 Python 脚本**绕开。

```bash
cd backend
venv\Scripts\python scripts\init_admin.py
# 自动检测 + ALTER 缺失字段（role/is_active/last_login + 索引）
# bcrypt 哈希密码 admin123
# 创建/更新 admin 账号
# 输出: id=a0000000-0000-0000-0000-000000000001, role=admin
```

> 重复运行是幂等的，会更新 admin 密码。改密码后重新登录即可。

### 3.3 启动后端

```bash
cd backend
python -m venv venv
# Windows 用 venv\Scripts\activate；Git Bash 用 venv/Scripts/activate
venv/Scripts/activate
pip install -r requirements.txt

# 启动（默认 8001）
uvicorn app.main:app --host 0.0.0.0 --port 8001

# 可选：加 --reload 自动重载
uvicorn app.main:app --reload --host 0.0.0.0 --port 8001
```

后端启动后访问：
- Swagger: http://localhost:8001/docs
- 健康检查: http://localhost:8001/health

### 3.4 启动前端

```bash
cd frontend
npm install   # 已配 .npmrc 走 npmmirror 国内镜像
npm run dev
```

前端启动后访问 http://localhost:3000

> ⚠️ 端口同步：前端默认连 8001。改后端端口要同步改 `frontend/src/lib/api.ts:5` 的 `BASE_URL`。

### 3.5 验证

打开浏览器：
1. 仪表盘 `/` 应该有统计数据
2. `/duihua` 输 "出 5 道搜索算法练习题" 应看到**逐字流式渲染**（验证 dual-format 协议）
3. `/profile` 应看到 6 维画像

## 4. 配置环境变量

后端 `.env`（`backend/.env`，**已 gitignore**）：

```env
# LLM 客户端（开发用 MiniMax）
MINIMAX_API_KEY=your_minimax_key
MINIMAX_BASE_URL=https://api.minimax.chat/v1
MINIMAX_MODEL=MiniMax-M3

# SPARK_BASE_URL=https://spark-api-open.xf-yun.com/v1
# SPARK_APP_ID=your_app_id

# 数据库
DATABASE_URL=postgresql+asyncpg://postgres:123456@localhost:5432/zhishu

# CORS（允许前端的地址）
CORS_ORIGINS=["http://localhost:3000"]
```

前端 `.env.local`（`frontend/.env.local`，**可选**，有默认值）：

```env

NEXT_PUBLIC_API_URL=http://localhost:8001/api/v1
```

## 5. 常用命令

### 5.1 服务管理

```bash
# 后端：直接 Ctrl+C 退出 uvicorn

# Windows 僵尸进程清理（如果端口被占但查不到 PID）：
powershell -Command "Get-NetTCPConnection -LocalPort 8001 -State Listen | Select-Object OwningProcess"
powershell -Command "Stop-Process -Id <PID> -Force"

# 端到端启动脚本（在项目根目录）：
# 后端
cd "d:/桌面/ZhiShu/ZhiShu/backend" && venv/Scripts/python -m uvicorn app.main:app --host 0.0.0.0 --port 8001

# 前端
cd "d:/桌面/ZhiShu/ZhiShu/frontend" && npx next dev
```

### 5.2 数据库管理

```bash
# 进入 PostgreSQL
psql -U postgres -d zhishu

# 备份数据库
pg_dump -U postgres zhishu > backup_$(date +%Y%m%d).sql

# 恢复数据库
psql -U postgres -d zhishu < backup_20260609.sql

# 清空所有表（危险！仅调试用）
psql -U postgres -d zhishu -c "TRUNCATE chat_messages, chat_sessions, learning_records, exercises, resources, learning_paths, document_chunks, student_profiles, students CASCADE;"
```

### 5.3 健康检查

```bash
# 后端
curl http://localhost:8001/health
# → {"status":"healthy"}

# 前端
curl -I http://localhost:3000
# → HTTP/1.1 200 OK
```

## 7. 故障排查

### 7.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 后端起不来：`Address already in use` | 8001 端口被占 | `powershell Stop-Process` 或换 8002 |
| 浏览器连不上后端 | 前端 8001 ≠ 后端端口 | 同步 `frontend/src/lib/api.ts:5` |
| 流式输出"卡住"几秒再一次性出 | 后端没装新 chat.py | 确认 `git log backend/app/api/chat.py` 有 2026-06-09 dual-format 修复 |
| LLM 返回 `401 Unauthorized` | API key 错误 | 检查 `.env` 里的 `MINIMAX_API_KEY` 或 `SPARK_API_KEY` |
| LLM 返回 `<think>` 标签污染输出 | 模型默认带推理标签 | 确认 chat.py:42-62 的 `_strip_think` 函数存在 |
| pgvector 报 `type "vector" does not exist` | PostgreSQL 扩展未装 | 当前用 JSONB fallback 方案，**忽略此错误** |
| 数据库连接 `FATAL: password authentication failed` | 密码错 | 确认 `DATABASE_URL` 里的 `123456` 与 `init_db.sql` 一致 |
| 前端 404 路由 | `.next` 缓存损坏 | `Remove-Item frontend/.next -Recurse` → 重启 `npm run dev` |

### 7.2 性能调优

- 后端并发：默认 uvicorn 单进程，可加 `--workers 4`
- 数据库连接池：检查 `backend/app/core/database.py` 的 `pool_size`（默认可能太低）
- LLM 流式：`temperature=0.7`（创作）/ `0.1`（路由分类）
- 前端：`next build` + `next start` 走 production 模式，比 `dev` 快 5-10x

## 8. 备份与恢复

### 8.1 备份脚本（Windows PowerShell）

```powershell
# backup.ps1
$date = Get-Date -Format "yyyyMMdd"
$backupDir = "D:\backups\zhishu"
New-Item -ItemType Directory -Force -Path $backupDir

# 备份数据库
pg_dump -U postgres zhishu > "$backupDir\db_$date.sql"

Write-Host "备份完成: $backupDir\db_$date.sql"
```

### 8.2 恢复

```bash
psql -U postgres -d zhishu < backup_20260609.sql
```
