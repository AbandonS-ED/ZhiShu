## 3. 快速部署

### 3.1 克隆代码

```bash
git clone https://github.com/your-repo/zhishu.git
cd zhishu
```

### 3.2 配置环境变量

```bash
cp .env.example .env
```

编辑 `.env` 文件：

```env
# 讯飞星火API
SPARK_APP_ID=your_app_id
SPARK_API_KEY=your_api_key
SPARK_API_SECRET=your_api_secret

# 数据库
POSTGRES_DB=zhishu
POSTGRES_USER=postgres
POSTGRES_PASSWORD=your_password

# Redis
REDIS_URL=redis://redis:6379/0

# MinIO
MINIO_ROOT_USER=minioadmin
MINIO_ROOT_PASSWORD=your_password

# JWT
JWT_SECRET=your_jwt_secret
```

### 3.3 启动服务

```bash
# 构建并启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

### 3.4 初始化数据库

```bash
# 初始化数据库表
docker-compose exec api-gateway python -m app.db.init

# 导入知识库数据
docker-compose exec api-gateway python -m app.scripts.init_knowledge_base
```

### 3.5 访问系统

- 前端：http://localhost:3000
- API文档：http://localhost:8000/docs
- MinIO控制台：http://localhost:9001

## 4. Docker Compose配置

### 4.1 docker-compose.yml

```yaml
version: '3.8'

services:
  # 前端
  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    ports:
      - "3000:3000"
    environment:
      - NEXT_PUBLIC_API_URL=http://api-gateway:8000
    depends_on:
      - api-gateway
    restart: unless-stopped

  # API网关
  api-gateway:
    build:
      context: ./backend
      dockerfile: Dockerfile
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=postgresql://postgres:password@postgres:5432/zhishu
      - REDIS_URL=redis://redis:6379/0
      - SPARK_APP_ID=${SPARK_APP_ID}
      - SPARK_API_KEY=${SPARK_API_KEY}
      - SPARK_API_SECRET=${SPARK_API_SECRET}
      - MINIO_ENDPOINT=minio:9000
      - MINIO_ACCESS_KEY=${MINIO_ROOT_USER}
      - MINIO_SECRET_KEY=${MINIO_ROOT_PASSWORD}
    depends_on:
      - postgres
      - redis
      - minio
    restart: unless-stopped

  # Celery Worker
  celery-worker:
    build:
      context: ./backend
      dockerfile: Dockerfile
    command: celery -A app.core.celery worker --loglevel=info
    environment:
      - DATABASE_URL=postgresql://postgres:password@postgres:5432/zhishu
      - REDIS_URL=redis://redis:6379/0
    depends_on:
      - postgres
      - redis
    restart: unless-stopped

  # PostgreSQL + pgvector
  postgres:
    image: pgvector/pgvector:pg16
    ports:
      - "5432:5432"
    environment:
      - POSTGRES_DB=zhishu
      - POSTGRES_USER=postgres
      - POSTGRES_PASSWORD=password
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql
    restart: unless-stopped

  # Redis
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    restart: unless-stopped

  # MinIO
  minio:
    image: minio/minio
    ports:
      - "9000:9000"
      - "9001:9001"
    environment:
      - MINIO_ROOT_USER=minioadmin
      - MINIO_ROOT_PASSWORD=minioadmin
    command: server /data --console-address ":9001"
    volumes:
      - minio_data:/data
    restart: unless-stopped

volumes:
  postgres_data:
  redis_data:
  minio_data:
```

## 5. 常用命令

### 5.1 服务管理

```bash
# 启动所有服务
docker-compose up -d

# 停止所有服务
docker-compose down

# 重启单个服务
docker-compose restart api-gateway

# 查看服务日志
docker-compose logs -f api-gateway

# 进入容器
docker-compose exec api-gateway bash
```

### 5.2 数据库管理

```bash
# 进入PostgreSQL
docker-compose exec postgres psql -U postgres -d zhishu

# 备份数据库
docker-compose exec postgres pg_dump -U postgres zhishu > backup.sql

# 恢复数据库
cat backup.sql | docker-compose exec -T postgres psql -U postgres -d zhishu
```

### 5.3 Redis管理

```bash
# 进入Redis CLI
docker-compose exec redis redis-cli

# 清空缓存
docker-compose exec redis redis-cli FLUSHALL
```

## 7. 故障排查

### 7.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| API服务启动失败 | 数据库未就绪 | 等待数据库启动后重启 |
| 向量检索失败 | pgvector扩展未安装 | 检查init.sql是否执行 |
| 讯飞API调用失败 | API密钥错误 | 检查.env配置 |
| 前端无法访问 | API服务未启动 | 检查api-gateway状态 |

### 7.2 查看错误日志

```bash
# 查看API错误
docker-compose logs api-gateway | grep ERROR

# 查看数据库错误
docker-compose logs postgres | grep ERROR
```

## 8. 备份与恢复

### 8.1 备份脚本

```bash
#!/bin/bash
# backup.sh

DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="./backups"

mkdir -p $BACKUP_DIR

# 备份数据库
docker-compose exec -T postgres pg_dump -U postgres zhishu > $BACKUP_DIR/db_$DATE.sql

# 备份MinIO数据
docker-compose exec -T minio mc mirror /data $BACKUP_DIR/minio_$DATE/

echo "备份完成: $BACKUP_DIR"
```

### 8.2 恢复脚本

```bash
#!/bin/bash
# restore.sh

BACKUP_FILE=$1

# 恢复数据库
cat $BACKUP_FILE | docker-compose exec -T postgres psql -U postgres -d zhishu

echo "恢复完成"
```
