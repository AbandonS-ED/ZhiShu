| 项目 | 内容 |
|------|------|
| **测试ID** | TC-F2-003 |
| **测试目标** | 验证 Exercise Agent 生成多类型练习题 + ⭐ **dual-format 真流式输出** |
| **前置条件** | 已有学生画像；后端 chat.py 是 2026-06-09 之后的版本（含 `STREAM_EXERCISE_SYSTEM` + `_strip_think`） |
| **测试步骤** | 1. 在 `/duihua` 输 "出 5 道搜索算法练习题"<br>2. **观察流式事件序列**：`session → progress → token* → result → done`<br>3. **观察 token 内容**：是 markdown 格式（"📝 为你生成了 5 道题…"），**不是 JSON 源码**<br>4. **观察 token 频率**：**逐字/逐行**出现，**不是等几秒再一次性出现**<br>5. 完成后右侧"已生成资源"应出现 "搜索算法 · 练习题 · 完成" |
| **预期结果** | 1. 至少 5 道 `type=token` 事件（每题 1 个）<br>2. token 内容是 markdown，非 `{"exercises":[...]}` JSON<br>3. 响应时间 < 30s<br>4. `type=result` 事件含 `data.exercises` 数组（≥ 5 项）<br>5. 前端 `markdownToHtml()` 渲染正常 |
| **优先级** | **P0**（2026-06-09 新增，本项目的关键体验特性） |
| **自动化** | `curl -N -X POST http://localhost:8001/api/v1/chat/stream -H 'Content-Type: application/json' -d '{"student_id":"...","message":"出 5 道搜索算法练习题"}'` 应看到连续的 `data: {"type":"token",...}` 事件 |

#### TC-F2-004: ⭐ dual-format 协议 separator 检测

| 项目 | 内容 |
|------|------|
| **测试ID** | TC-F2-004 |
| **测试目标** | 验证 LLM 输出包含 `---JSON_DATA---` 分隔符 + 后端正确分割 |
| **前置条件** | TC-F2-003 通过 |
| **测试步骤** | 1. 同 TC-F2-003 抓取完整 SSE 流<br>2. 检查 token 事件：前 N 个是 markdown，遇到分隔符后**停止 emit token**<br>3. 检查 result 事件：含 `data.exercises` 数组，**与 markdown 部分的题目一一对应** |
| **预期结果** | 1. 用户看不到 JSON 源码（因为 `sep_found=True` 后停止 emit）<br>2. JSON 解析成功（`parse_json_response` 兜底）<br>3. JSON 中的题目数 ≥ markdown 中看到的题目数 |
| **优先级** | P0 |
| **回归保护** | 如果 LLM 不遵循 dual-format（无分隔符），fallback 走 `parse(stream_text)` 应仍能解析整段 |
